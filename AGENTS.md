# Development Rules

## Branching
- main is protected (no direct push, no force-push)
- Work on topic branches and merge via PR
- Branch prefixes:
  - feature/* : new functionality
  - fix/*     : bug fixes
  - add/*     : additions (deps/docs/etc)
  - remove/*  : removals
  - chore/*   : maintenance

## Commit Messages
- Use a prefix: `feat:`, `add:`, `remove:`, `fix:`, `chore:`
